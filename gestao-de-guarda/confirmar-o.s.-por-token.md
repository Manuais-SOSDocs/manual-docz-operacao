# Confirmar O.S. por Token

Permitir que um colaborador registre a **confirmação do atendimento de uma solicitação**, validando a ordem de serviço (O.S) com um **token** fornecido pelo usuário final. Também permite reportar pendências.

<figure><img src="../.gitbook/assets/image (266).png" alt=""><figcaption></figcaption></figure>

### Confirmando o Token:

{% stepper %}
{% step %}
Digite o número da ordem de serviço (identificador da solicitação).

<figure><img src="../.gitbook/assets/image (323).png" alt=""><figcaption></figcaption></figure>
{% endstep %}

{% step %}
Informe o Token de validação da O.S informado pelo cliente/usuário final.

<div align="left"><figure><img src="../.gitbook/assets/image (324).png" alt=""><figcaption></figcaption></figure></div>
{% endstep %}

{% step %}
Marque essa opção para validar que o atendimento ocorreu com a participação e ciência do usuário final.

<div align="left"><figure><img src="../.gitbook/assets/image (325).png" alt="" width="563"><figcaption></figcaption></figure></div>
{% endstep %}

{% step %}
Marque se houveram falhas, atrasos ou qualquer ocorrência fora do planejado.

<div align="left"><figure><img src="../.gitbook/assets/image (326).png" alt="" width="406"><figcaption></figcaption></figure></div>
{% endstep %}

{% step %}
Caso a opção anterior seja marcada, descreva aqui os problemas/pontos de atenção.

<figure><img src="../.gitbook/assets/image (327).png" alt=""><figcaption></figcaption></figure>
{% endstep %}

{% step %}
Clique em  ![](<../.gitbook/assets/image (328).png>).
{% endstep %}
{% endstepper %}

{% hint style="success" %}
**Ação concluída!**
{% endhint %}
